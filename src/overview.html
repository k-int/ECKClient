Provides the classes necessary to interface with the ECKCore using a java 7 client

All interfaces to the ECK functionality are static methods with each module having its own interface

  • {@link com.k_int.euinside.client.module.dataTransformation.Transform Data Transformation}
  • {@link com.k_int.euinside.client.module.metadataDefinition.Definition Metadata Definition}
  • {@link com.k_int.euinside.client.module.aggregator.EuropeanaEnrichments Europeana Enrichments}
  • {@link com.k_int.euinside.client.module.aggregator.EuropeanaStatistics Europeana Statistics}
  • {@link com.k_int.euinside.client.module.pidgenerate.PIDGenerate PID Generation}
  • {@link com.k_int.euinside.client.module.preview.Preview Preview}
  • {@link com.k_int.euinside.client.module.setmanager.SetManager Set Manager}
  • {@link com.k_int.euinside.client.module.statistics.Statistics Statistics}
  • {@link com.k_int.euinside.client.module.validation.Validate Validation (Alternates between monguz and semantika)}
  • {@link com.k_int.euinside.client.module.validation.ValidateMonguz Validation (monguz)}
  • {@link com.k_int.euinside.client.module.validation.ValidateSemantika Validation (semantika)}
  • {@link com.k_int.euinside.client.module.sword.SWORDPush SWORD v1 client}

Before a call is made to any module the base url of the ECKCore module needs to be set, this is achieved by calling BaseModule.setCoreBaseURL and passing the base url to the core as the parameter.

If this call is not made then all subsequent will fail as there is no default value for the core base url.

Examples:

1. To generate a PID
BaseModule.setCoreBaseURL("http://euinside.k-int.com/ECKCore2"); // Needs to be done once in the program, before any other calls to the ECKClient
String pid = PIDGenerate.generate("institutionURL", "item", "123456");

2. To validate a lido record
BaseModule.setCoreBaseURL("http://euinside.k-int.com/ECKCore2"); // Needs to be done once in the program, before any other calls to the ECKClient
ValidationResult result = Validate.sendFiles("providerCode", "myLidoFile.xml");

3. To retrieve the status of a set
BaseModule.setCoreBaseURL("http://euinside.k-int.com/ECKCore2"); // Needs to be done once in the program, before any other calls to the ECKClient
Status status = SetManager.getStatus("provider", "set");

4. To convert from LIDO to EDM
BaseModule.setCoreBaseURL("http://euinside.k-int.com/ECKCore2"); // Needs to be done once in the program, before any other calls to the ECKClient
byte [] edmRecord = Transform.transformWait(providerCode, batchId, Format.LIDO, Format.EDM, lidoRecord);

5. To record a statistic in the statistics module
BaseModule.setCoreBaseURL("http://euinside.k-int.com/ECKCore2"); // Needs to be done once in the program, before any other calls to the ECKClient
Statistics.update("SetManager", "Update", startTime, endTime, numberFailed, numberSuccessful);
If you are processing a batch of records, then the classes {@link com.k_int.euinside.client.module.statistics.Tracker Tracker} or {@link com.k_int.euinside.client.module.statistics.TrackerBatch TrackerBatch} maybe more suitable for you
