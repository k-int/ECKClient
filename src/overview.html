Provides the classes necessary to interface with the ECKCore using a java 7 client

All interfaces to the ECK functionality are static methods with each module having its own interface

  • {@link com.k_int.euinside.client.module.metadataDefinition.Definition Metadata Definition}
  • {@link com.k_int.euinside.client.module.pidgenerate.PIDGenerate PID Generation}
  • {@link com.k_int.euinside.client.module.preview.Preview Preview}
  • {@link com.k_int.euinside.client.module.setmanager.SetManager Set Manager}
  • {@link com.k_int.euinside.client.module.validation.Validate Validation (monguz)}
  • {@link com.k_int.euinside.client.module.validation.Validate2 Validation (semantika)}

Before a call is made to any module the base url of the ECKCore module needs to be set, this is achieved by calling BaseModule.setCoreBaseURL and passing the base url to the core as the parameter.

If this call is not made then all subsequent will fail as there is no default value for the core base url.

Examples:

1. To generate a PID
BaseModule.setCoreBaseURL("http://euinside.k-int.com/ECKCore2"); // Needs to be done once in the program, before any other calls to the ECKClient
String pid = PIDGenerate.generate("institutionURL", "item", "123456");

2. To validate an edm record using the monguz validator
BaseModule.setCoreBaseURL("http://euinside.k-int.com/ECKCore2"); // Needs to be done once in the program, before any other calls to the ECKClient
Errors errors = Validate2.sendFiles("myLidoFile.xml");

3. To retrieve the status of a set
BaseModule.setCoreBaseURL("http://euinside.k-int.com/ECKCore2"); // Needs to be done once in the program, before any other calls to the ECKClient
Status status = SetManager.getStatus("provider", "set");
